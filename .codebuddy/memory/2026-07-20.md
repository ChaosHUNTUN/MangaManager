# 2026-07-20

## 项目审计

### 🔴 严重 (Critical)

1. **EHentai Cookie 可能硬编码** — `ehentai_cookie.txt` 如被提交到 Git 则泄露凭据
2. **AlbumConfig.Gids 与 LocalGallery.AlbumKey 不同步** — `Save()` 直接覆盖 Gids，`AutoAssignToAlbumsAsync` 写入 AlbumKey 但未更新 Gids，存在静默丢失风险
3. **N+1 查询**: `BatchTagController.BatchAddTags` 嵌套循环逐条查 `AnyAsync`；`MangaService.ApplyTagsAsync` 循环中逐个 `SaveChanges`
4. **前端 `api.js` 核心 `request` 函数无超时/重试/请求去重**
5. **前端 `Detail.jsx` L56 用了未导入的 `useRef`** — 运行时错误

### 🟠 高 (High)

6. **数据源头不一致**: `.meta.json` vs `local_gallery` 表，`UpdateMetaTagsAsync` 只写文件不更新 DB
7. **缺失索引**: `manga.FolderPath`、`manga.Title`、`local_gallery.Title`、`scan_log.Status`
8. **前端重复 API 请求**: `loadPaged` useEffect 依赖过多，`loadMetas` 非 useCallback 导致频繁重注册
9. **前端内存泄漏**: `EHentai.jsx` 初始化 effect 无 AbortController 清理
10. **DownloadManager 状态转换**: 部分下载完成被标记为 completed 而非 partial，无法恢复

### 🟡 中 (Medium)

11. **SQLite 并发写入限制**: 多线程同时写会抛 "database is locked"
12. **AlbumConfig.Count 冗余列**: 可从 Gids 计算，需手动同步
13. **FileSystemWatcher 事件丢失风险**: 无定期全量补偿同步
14. **EHentai.jsx `loadMore` 无竞态保护**: 快速滚动可能重复请求

### 🟢 低 (Low)

15. **表命名不一致**: snake_case vs PascalCase
16. **前端 `Reader.jsx` 6 个 useEffect 仅同步 ref** — 可合并
17. **前端翻译批处理依赖 `galleryMetas.length`** — 内容变长度不变时翻译不更新
