# 2026-06-17

## 本地画廊查询性能优化

### 问题
本地画廊打开速度过于缓慢，每次翻页/加载封面都触发全量磁盘 I/O。

### 已实施优化

**后端 LocalGalleryService.cs:**
1. `ScanLocalGalleriesInternal` 缓存从 5 秒延长至 **30 秒**，大幅减少翻页时的全量磁盘扫描次数
2. 新增 `InvalidateScanCache()` 公开方法，在删除/导入/重下载/编辑标签后主动失效缓存
3. 新增 `GetCachedItem(gid)` O(1) 查找（基于 gid→item 索引）
4. 新增 `GetCachedPageFiles(gid)` 按 gid 缓存文件列表（10 秒 TTL），避免 `GetPageFilePath` 和 `GetGalleryPages` 每次重复枚举目录
5. `GetDetailAsync` 合并两段重复的 meta.json 读取为一次

**后端 LocalGalleryController.cs:**
6. `GetCover` 改为使用 `_svc.GetCachedItem()` 从扫描缓存获取封面路径，不再独立做 Directory.GetDirectories + Directory.GetFiles（之前每张封面都是一次独立磁盘扫描）
7. `DeleteGallery`/`Redownload`/`BatchRedownload`/`ImportGallery`/`BatchImport`/`UpdateMetaTags` 操作后调用 `InvalidateScanCache()`

**前端 LocalGallery.jsx:**
8. 修复 `loadGalleries()` 未定义 bug（3 处：导入、批量导入、编辑标签保存），改为 `loadMetas(); loadPaged()`
9. **合并两个 useEffect 为一个**：原来 `[activeGroup,search,sortBy,pageSize,albumsLoaded]` 和 `[page,albumsLoaded]` 两个 effect 在 `albumsLoaded` 变为 true 时同时触发 `loadPaged(1)`，造成 `/api/local/galleries/paged` 被调用 2 次。合并为 `[activeGroup,search,sortBy,pageSize,page,albumsLoaded]` 统一 effect。
10. **翻译防重复**：翻译 useEffect 依赖 `galleryMetas.length`，在 StrictMode 双重挂载时会触发两次。添加 `translateTriggerRef` 记录已翻译的长度，相同长度跳过。
11. **修复阅读器退出后丢失筛选状态**：`ReaderLocal.jsx` 中三处返回导航都用的 `navigate('/')`（Esc键、空图片页面、顶部返回按钮），跳转到光秃秃的根路径丢掉了所有 URL 查询参数。改为 `navigate(-1)`（浏览器后退），URL 完整保留。
12. **修复随机模式退出阅读器后丢失**：`loadRandom()` 清空了所有 URL 参数导致退回到 `/local` 无任何状态。改为在 URL 中保留 `random=true` 标记，组件挂载/恢复时检测该参数自动重新加载随机结果。同时修改 `updateParams`：任何非随机操作自动清除 `random` 标记，避免随机模式"粘滞"。随机按钮改为只设置 URL 参数（`random=true`），由统一的 useEffect 处理加载，避免 `loadRandom` 被按钮直接调用 + useEffect 双重触发。
13. **修复阅读器 Esc/返回按钮在切换画廊后退错位置**：`navigate(-1)` 依赖浏览器历史，在阅读器内切换画廊（上一部/下一部）后变成退回上一部作品而非退出。改为：进入阅读器时保存 `window.location.search` 到 sessionStorage（`reader-local-return-url`），Esc 和返回按钮读取它拼出 `/local?params` 精确导航回去。
14. **在线画廊默认勾选里站+热门**：`exhentai` 默认值 `false→true`，`currentExRef` 同步 `false→true`；`popularMode` 默认值 `false→true`。初始化 useEffect 添加 `fetchEHGalleries('', 0, true, null, { popular: true })` 调用，打开页面即显示里站热门内容。
15. **专辑配置新增 Count/KeyTag 字段 + 查询接口**：
    - Entity：`AlbumConfig` 新增 `Count`(int,默认0) 和 `KeyTag`(string?,EH标准标签如"artist:haiboku")
    - 迁移：`20260618140000_AddAlbumCountAndKeyTag`（含已存数据 Count=json_array_length(Gids)），已通过 `dotnet ef database update` 应用到数据库
    - 删除重复旧迁移 `20260618071344`
    - Save()：自动 `Count = gids.Length`，Key 含冒号自动设为 KeyTag
    - 新端点：`GET /api/albums/summary`（key,name,color,count,createdAt）、`GET /api/albums/{key}/detail`（+gids+keyTag）
    - 前端：新增 `fetchAlbumSummary()`、`fetchAlbumDetailV2()` 到 api.js
16. **画廊元数据 DB 化**：
    - Entity：新增 `LocalGallery` 实体（Gid/TITLE/DirPath/Category/Language/Rating/FileCount/FileSize/CoverFile/Artists/Groups/OnlineUrl/Token/DownloadedAt/LastModified/SyncedAt），表名 `local_gallery`
    - 迁移：`20260618145000_AddLocalGalleryTable`（含索引：Category/Language/DownloadedAt/LastModified）
    - 后台同步：`GallerySyncService` (BackgroundService)，启动全量扫描目录→写入DB，FileSystemWatcher 实时增量同步，5分钟一致性检查
    - Service 改造：`LocalGalleryService` 注入 `IServiceScopeFactory`，`GetPagedGalleries`/`GetGalleryMetas`/`GetGalleryGroups`/`GetRandomGalleries`/`GetCachedItem` 全部改为 EF LINQ 查询，移除文件扫描和内存缓存
    - 旧方法清理：删除 `ScanLocalGalleriesInternal`（静态缓存）、`ApplyFilters`、`ApplySort`、`MatchBasic` 等已废弃的内存方法
17. **下载断点续传**：`DownloadManager.LoadTasksFromDb` 中将中断任务从标记 `"failed"` 改为标记 `"pending"` + 自动入队。`DownloadTaskAsync` 已内置读取 `.progress` 文件跳过已下载页面，重启后自动从断点恢复，无需从头下载。
18. **修复 WPF 控制台在服务重启时卡死**：
    - 根因：`DispatcherTimer` tick 在 UI 线程上做 HTTP 调用，后端停止时 TCP 连接超时堆积导致消息泵阻塞
    - `_http` 改用 `SocketsHttpHandler { ConnectTimeout = 2s }` 限制 TCP 连接超时
    - 新增 `_timersPaused` 标志：`StopApi` 时立即设为 true 阻止所有轮询；`StartApiAsync` 确认端口就绪后恢复
    - `RefreshDownloadTasks`/`RefreshServiceStatus`/`IsPortOpen` 包装在 `Task.Run` 中移出 UI 线程
    - `BtnApiRestart_Click` 改为 `await StartApiAsync()` 避免 fire-and-forget
