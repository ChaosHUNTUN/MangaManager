## ExHentai 里站连通性与跳转优化

### 修改内容
1. **后端 DTO** (`EhentaiService.cs`):
   - `GalleryListResult` 新增 `IsExhentai` 字段
   - `GalleryItem` 新增 `IsExhentai` 字段
   - `GalleryDetail` 新增 `IsExhentai` 字段

2. **后端 Service** (`EhentaiService.cs`):
   - `GetGalleriesAsync`: 返回结果中标记 `IsExhentai = exhentai`
   - `ParseList`: 每个 `GalleryItem` 根据 `host == HOST_EX` 设置 `IsExhentai`
   - `GetGalleryDetailAsync`: 先尝试表站，若返回 "This gallery is unavailable" 则自动尝试里站，并设置 `isExhentai` 标记

3. **后端 Controller** (`EhentaiController.cs`):
   - `/api/ehentai/galleries` 响应新增 `r.IsExhentai` 字段
   - `/api/ehentai/connectivity` 增强：同时检测表站(e-hentai.org)和里站(exhentai.org)连通性，返回 `eReachable` 和 `exReachable`

4. **前端 GalleryDetail.jsx** (本地画廊详情弹窗):
   - "在 E-Hentai 查看"链接根据 `detail.isExhentai` 切换 `e-hentai.org` 或 `exhentai.org`
   - 按钮文字也动态显示 "ExHentai" 或 "E-Hentai"

5. **前端 EHentai.jsx** (在线画廊详情弹窗):
   - "在 E-Hentai 打开"链接根据 `detail.isExhentai` 切换域名
   - 按钮文字动态显示

## 本地画廊筛选状态保持
- 问题：筛选→进入阅读→返回画廊，筛选条件全部丢失（useState 在组件卸载时重置）
- 修复 (`LocalGallery.jsx`):
  - `saveGalleryState()`: 在 `openDetail()` 和 GalleryCard 阅读按钮 onClick 中，将 search/sortBy/pageSize/page/viewMode/activeGroup 保存到 sessionStorage
  - `restoreState()`: 在组件初始化时读取 sessionStorage 恢复状态，恢复后立即清除 key 避免误恢复
  - 使用 useCallback 确保 saveGalleryState 引用稳定
- 修改文件: LocalGallery.jsx（新增 saveGalleryState/restoreState）、GalleryDetail.jsx（无需额外改动）
