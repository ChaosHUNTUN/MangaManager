## Bug 修复：EF Core Migration baseline 检测失败
- **问题**：`ExecuteSqlRaw("SELECT COUNT(*) ...")` 对 SELECT 语句永远返回 0（受影响行数），导致已有数据库未被识别，`Migrate()` 尝试创建已存在的表 → `SQLite Error 1: table already exists`
- **修复**：改用 ADO.NET `DbConnection.CreateCommand().ExecuteScalar()` 获取 `COUNT(*)` 的真实值
- 改用 `GetDbConnection()` 直接操作底层连接，baseline 创建也改用 `ExecuteNonQuery()` 保持一致
- 添加 `Microsoft.EntityFrameworkCore.Infrastructure` using（`GetDbConnection` 扩展方法需要）

## Bug 修复：baseline 检测二次失败
- 第一次崩溃后 `__EFMigrationsHistory` 空表已创建（无记录），第二次启动时 `hasHistoryTable`=true 但 `hasHistoryRecords` 未检查 → 再次跳过 baseline → 再次崩溃
- 修复：增加 `SELECT COUNT(*) FROM __EFMigrationsHistory` 检查，空表也视为需要 baseline

## Bug 修复：前端专辑筛选失效
- **根因**：`handleSelectGroup` 等函数中连续多次调用 `setActiveGroup`/`setSortBy`/`setPage`，每个都独立触发 `setSearchParams(prev => ...)`。虽然 React 18 批处理，但每个调用创建新的 `URLSearchParams(prev)`，后一次调用可能覆盖前一次的结果
- **修复**：将所有多参数更新合并为单次 `updateParams({...})` 调用，一次 `setSearchParams` 更新所有 URL 参数
- 影响范围：`handleSelectGroup`、`handleCreateAlbum`、`applySearchTag`、排序/分页 select、自动分组标签点击、"全部"按钮

## Bug 修复：搜索框拼音输入法 IME 组合输入异常
- **问题**：使用拼音输入中文时，拼音字母（如 l、i、r）被 `onChange` 当作普通字符直接写入 URL，导致出现 `llirru` 等异常累增
- **根因**：`value={search}` 来自 URL query string，每次 `setSearchParams` 触发重渲染重置 input value，打断 IME composition 状态
- **第一次尝试（失败）**：使用 `isComposingRef` + `onCompositionStart/End` 手动管理，但 `compositionend` 事件在浏览器/输入法中的触发时机不一致，导致普通英文输入也无法正常工作
- **最终修复**：使用 `e.nativeEvent.isComposing` 属性（React 17+ 原生支持），在 `handleSearchInput` 开头检查 `if (e.nativeEvent.isComposing) return` — 这是浏览器原生暴露的 IME 状态，比手动管理 ref 更可靠
- 移除了 `isComposingRef`、`handleCompositionStart`、`handleCompositionEnd` 和对应的 `onCompositionStart/End` 绑定

## Bug 修复：E-Hentai 在线模块页面加载 400 错误
- **问题**：进入 E-Hentai 页面时，无论 Cookie 是否配置都立即发起 `GET /api/ehentai/galleries?page=0&exhentai=true&popular=true`，未配置 Cookie 时后端返回 400 Bad Request
- **后端修复** (`EhentaiController.cs`)：5 处 `!HasCookie()` 检查从 `return BadRequest(...)` 改为 `return Unauthorized(...)` — 401 才是"需要认证但未提供"的正确语义，消息统一为"请先配置并保存 E-Hentai Cookie"
- **前端修复** (`EHentai.jsx`)：
  - `useEffect` 初始化改为 `async init()`：先 `await loadCookie()`，不再在 Cookie 检查完成前发请求
  - 移除无 Cookie 时的默认列表加载（不再静默发 401 请求）
  - 新增 Cookie 未配置的友好提示 UI（红色边框卡片，引导用户点击配置按钮）
  - 保存 Cookie 成功后自动加载热门列表并清除之前的 error
  - 空状态提示改为区分"未配置 Cookie"和"已配置但无结果"两种情况
