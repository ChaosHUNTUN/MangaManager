# 本地画廊管理增强方案

> 状态：设计提案 | 日期：2026-06-07

---

## 当前痛点

本地画廊数量多时，只有 4 列网格 + 无搜索/筛选/分页，难以快速找到目标作品。

---

## 方案设计

### 一、整体架构

```
下载时                           加载时
┌──────────┐   写入    ┌──────────────────┐   读取   ┌──────────────┐
│ 下载完成  │ ──────→  │ .meta.json       │ ──────→ │ 前端分组展示  │
│          │          │ { tags, rating,  │         │ 按 Artist/   │
│          │          │   category, ... } │         │ Group 专辑   │
└──────────┘          └──────────────────┘         └──────────────┘
```

### 二、下载时写入元数据 (.meta.json)

下载完成后，在画廊目录写入 `{gid}-{title}/.meta.json`：

```json
{
  "gid": 1234567,
  "title": "Sample Gallery",
  "category": "doujinshi",
  "rating": 4.5,
  "fileCount": 42,
  "tags": {
    "artist": ["artist_a", "artist_b"],
    "group": ["group_x"],
    "parody": ["original"],
    "character": [],
    "language": ["chinese"],
    "female": [],
    "male": []
  },
  "downloadedAt": "2026-06-07T12:00:00Z"
}
```

**后端改动**：
- `DownloadManager` 或 `EhentaiService` 下载完成后，获取 `GalleryDetail`（含 tags），序列化为 `.meta.json` 写入画廊目录
- `LocalGalleryItem` 新增 `Artist`、`Group`、`Category`、`Rating` 字段
- `ScanLocalGalleries` 扫描时读取 `.meta.json`，提取标签信息

### 三、前端专辑分组视图

#### 默认视图：按 Artist/Group 分组

```
┌──────────────────────────────────────────────────────────┐
│  📁 本地画廊 [128部]    │  🔍 [搜索标题/GID...]        │
├──────────────────────────────────────────────────────────┤
│  分组: [👤 按作者] [全部] [artist_a (12)] [artist_b (8)] │
│        [group_x (15)] [多作者 (5)] [📦 未分类 (10)]     │
├──────────────────────────────────────────────────────────┤
│  排序: [最近修改 ▼]  视图: [▦网格] [☰列表]  每页:[50▼] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ▓▓ artist_a (12部)                                      │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐                            │
│  │封面│ │封面│ │封面│ │封面│   ← 该 artist 下的画廊网格   │
│  │标题│ │标题│ │标题│ │标题│                              │
│  └────┘ └────┘ └────┘ └────┘                            │
│                                                          │
│  ▓▓ group_x (15部)                                       │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐                            │
│  │ ... │ │ ... │ │ ... │ │ ... │                          │
│  └────┘ └────┘ └────┘ └────┘                            │
│                                                          │
│  ▓▓ 📦 未分类 (10部)                                     │
│  ┌────┐ ┌────┐ ...                                       │

└──────────────────────────────────────────────────────────┘
```

#### 分组规则

| 分组 | 条件 |
|------|------|
| `artist_xxx` | tags.artist 中只有 1 个 artist |
| `group_xxx` | tags.group 中只有 1 个 group |
| `多作者` | artist + group 标签总数 > 1 |
| `未分类` | artist 和 group 都为空 |

#### 顶部快捷标签

点击标签切换到对应分组，只显示该分组下的画廊。支持水平滚动。

### 四、搜索与筛选（P0）

| 功能 | 实现 |
|------|------|
| **搜索框** | 输入关键词实时过滤标题/GID |
| **页数范围** | 全部 / 短篇(<30P) / 中篇(30-100P) / 长篇(>100P) |

### 五、排序选项（P0）

| 排序方式 | 说明 |
|---------|------|
| 最近修改（默认） | 按文件系统最后修改时间倒序 |
| 标题 A-Z | 按标题字母排序 |
| 页数最多/最少 | 按文件数量排序 |
| 大小最大/最小 | 按文件总大小排序 |

### 六、视图模式（P1）

| 模式 | 布局 |
|------|------|
| **网格**（默认） | 4 列卡片 |
| **列表** | 单列紧凑行：封面(80px) + 标题 + 页数 + 大小 |

### 七、分页（P1）

- 每页 50 部，可选 25 / 50 / 100
- 底部页码导航

---

## 实现计划

| 阶段 | 内容 | 涉及文件 |
|------|------|---------|
| **阶段 1** | 下载时写入 `.meta.json` | EhentaiService / DownloadManager |
| **阶段 2** | ScanLocalGalleries 读取元数据 | LocalGalleryService, LocalGalleryItem |
| **阶段 3** | 前端专辑分组视图 + 搜索/排序/分页/列表 | LocalGallery.jsx |

### 向后兼容

- 旧画廊（无 `.meta.json`）自动归入"未分类"组
- 可提供"补全元数据"按钮，对未分类画廊批量获取 E-Hentai 详情
